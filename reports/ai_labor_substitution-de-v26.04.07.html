Data Architecture Design

Defines the end:to:end pipeline structure using an adaptive medallion architecture. Data traverses three purpose:built layers : Bronze (raw ingestion), Silver (cleansed and conformed), and Gold (aggregated and analytics:ready) : ensuring separation of concerns, full data traceability, and downstream reliability at scale.

Selected Architecture Pattern

The pipeline architecture is automatically determined by analyzing source data signals : volume, source type, column composition, PII sensitivity, and duplicate rate : in conjunction with the declared business purpose. This adaptive selection eliminates manual configuration and ensures architectural alignment with actual use:case requirements.

Selected Pattern MEDALLION   MEDIUM CONFIDENCE
What It MeansBronze (raw) → Silver (cleaned/conformed) → Gold (aggregated/business:ready)
Best Used ForAnalytics, BI dashboards, reporting, large datasets with complex transformations

Why this pattern was selected:

  • Small dataset (1,230 rows) → Simple ETL may be sufficient
  • 9 numeric columns → aggregation in Gold layer adds value
  • Null values present → Silver cleaning layer recommended
  • Duplicate rate 2.28% → deduplication in Silver layer needed

Pattern scores (higher = better fit):

MEDALLION (3) SIMPLE (2) LAMBDA (0) KAPPA (0)

How scores are calculated: each pattern earns points based on signals in your data and business purpose.
Medallion: earns +1 per analytics keyword matched (dashboard, roi, revenue, campaign, kpi...) + points for PII columns, null values, numeric columns, date columns, or database source.
Simple: earns +2 for small datasets (under 5,000 rows).
Lambda: earns points when both batch and streaming keywords are detected.
Kappa: earns points for pure streaming sources (Kafka) or streaming:only keywords.
The pattern with the highest total score wins automatically.
